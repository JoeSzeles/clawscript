Introduction

ClawScript is a domain-specific language (DSL) designed for writing automated trading strategies, AI-powered analysis, and agent orchestration workflows. It compiles to JavaScript strategy classes that integrate with the Trade Claw Engine.

Key Features

  • Human-readable trading commands (BUY, SELL, EXIT)
  • Built-in technical indicators (RSI, EMA, MACD, Bollinger, etc.)
  • AI integration (AI_QUERY, RUN_ML)
  • Data fetching from web, social media, PDFs
  • Agent orchestration and multi-session communication
  • Error handling with TRY/CATCH
  • Visual flow builder with drag-and-drop
  • Compiles to executable JavaScript strategy classes

Architecture

  • Parser: clawscript-parser.cjs — Lexer + recursive-descent parser
  • AST: Typed node tree (Program → Statements)
  • Code Generator: AST → JavaScript strategy class
  • Editor UI: ig-clawscript-ui.js — Code editor with syntax highlighting
  • Flow Engine: ig-clawscript-flow.js — Visual node editor
  • Runtime: Strategy classes extend BaseStrategy

Quick Start

Here is a minimal ClawScript strategy that buys when RSI drops below 30:

// My first ClawScript strategy
DEF rsi = RSI(14)

IF rsi < 30 THEN
  BUY 1 AT MARKET STOP 20 REASON "RSI oversold"
ENDIF

This script:

  1. Defines an RSI indicator with period 14
  2. Checks if RSI is below 30 (oversold)
  3. Generates a BUY signal with size 1, market order, 20-point stop

Setup & Integration

Using the Editor

The ClawScript Editor is available in the IG Trading Dashboard under the ClawScript Editor tab. It provides:

  • A code editor with syntax highlighting and line numbers
  • A visual flow builder for drag-and-drop strategy creation
  • Real-time parsing and error feedback
  • Compile & Save to generate JavaScript strategy files

Using the Parser Programmatically

const { parseAndGenerate } = require('./clawscript-parser.cjs');

const code = 'DEF rsi = RSI(14)\nIF rsi < 30 THEN\n  BUY 1 AT MARKET\nENDIF';
const result = parseAndGenerate(code, 'MyStrategy');

console.log(result.ast);       // AST tree
console.log(result.js);        // Generated JavaScript
console.log(result.imports);   // ['indicators', 'trade']
console.log(result.variables); // ['rsi']

Strategy Integration

Compiled strategies are saved as .cjs files in the skills/bots/strategies/ directory. They extend BaseStrategy and are automatically discovered by the strategy loader.

Grammar Overview

ClawScript uses a line-oriented, keyword-driven grammar. Statements begin with a command keyword and are terminated by newlines or the start of the next command.

Program Structure

Program       ::= Statement*
Statement     ::= VarDecl | Trade | Exit | TrailStop | IfStatement
                | Loop | TryCatch | AICommand | DataCommand
                | AgentCommand | AdvancedCommand | FunctionDecl
                | FunctionCall | Chain | Assignment

Comments

// Single-line comment
/* Multi-line
   comment */

Literals

  • Numbers: 42, 3.14, 0.02
  • Strings: "hello" or 'hello'
  • Booleans: true, false
  • Null: null, nil

Operator Precedence (highest to lowest)

  1. Postfix: . (member access), [] (index)
  2. Unary: -, NOT / !
  3. Multiplicative: *, /, %
  4. Additive: +, -
  5. Crossover: CROSSES OVER, CROSSES UNDER
  6. Contains: CONTAINS
  7. Comparison: >, <, >=, <=, ==, !=
  8. Logical AND: AND / &&
  9. Logical OR: OR / ||

Trading Commands

BUY Trading

Generate a BUY signal at market, limit, or stop price.

Syntax: 
BUY <size> AT MARKET|LIMIT(<price>)|STOP [IF <condition>] [STOP <dist>] [LIMIT <dist>] [REASON <string>]
ParameterTypeRequiredDescription
sizenumber/exprNoPosition size (defaults to config size)
ATkeywordNoOrder type: MARKET, LIMIT(price), STOP
IFconditionNoConditional execution
STOPnumberNoStop distance in points
LIMITnumberNoLimit (take profit) distance
REASONstringNoTrade reason for logging
BUY 1 AT MARKET STOP 20 LIMIT 40 REASON "RSI oversold entry"
BUY 2 AT LIMIT(1850.50) STOP 15
BUY size AT MARKET IF rsi < 30 STOP 25
SELL Trading

Generate a SELL signal. Syntax identical to BUY.

SELL 1 AT MARKET STOP 20 LIMIT 40 REASON "RSI overbought"
SELLSHORT Trading

Open a short position. Same parameter structure as BUY/SELL.

SELLSHORT 1 AT MARKET STOP 30 REASON "Bearish divergence"
EXIT Trading

Close a position (all or partial).

Syntax: 
EXIT [ALL|PART(<size>)] [IF <condition>] [REASON <string>]
EXIT ALL REASON "Take profit hit"
EXIT PART(0.5) IF profit > 100 REASON "Partial close"
CLOSE Trading

Close the current position. Alias for EXIT ALL.

CLOSE REASON "End of day"
TRAILSTOP Trading

Set a trailing stop with optional acceleration parameters.

Syntax: 
TRAILSTOP <distance> [ACCEL <rate>] [MAX <max_rate>]
ParameterTypeDescription
distancenumberInitial trailing stop distance in points
ACCELnumberAcceleration factor (e.g., 0.02)
MAXnumberMaximum acceleration (e.g., 0.2)
TRAILSTOP 15 ACCEL 0.02 MAX 0.2

Variable Commands

DEF Variables

Define a constant variable. The value is computed once and cannot be reassigned.

DEF rsi_period = 14
DEF rsi = RSI(rsi_period)
DEF label = "my strategy"
DEF result = AI_QUERY "analyze this"
Note: DEF compiles to const in JavaScript. Use SET for mutable variables.
SET Variables

Set or update a mutable variable.

SET counter = 0
SET counter = counter + 1
SET sentiment = AI_QUERY "market mood?"
STORE_VAR Variables

Persist a variable to persistent storage. Survives restarts.

Syntax: 
STORE_VAR <key> <value> [GLOBAL]
STORE_VAR "last_trade_time" timestamp
STORE_VAR "total_trades" count GLOBAL
LOAD_VAR Variables

Load a variable from persistent storage.

Syntax: 
LOAD_VAR <key> [DEFAULT <value>]
DEF lastTime = LOAD_VAR "last_trade_time" DEFAULT 0

Control Flow Commands

IF / THEN / ELSE / ENDIF Control Flow

Conditional branching with optional ELSE clause. Supports ELSE IF chaining.

IF rsi < 30 THEN
  BUY 1 AT MARKET
ELSE IF rsi > 70 THEN
  SELL 1 AT MARKET
ELSE
  ALERT "RSI neutral" LEVEL "info"
ENDIF
LOOP / ENDLOOP Control Flow

Repeat a block N times or forever.

LOOP 5 TIMES
  ALERT "checking..."
  WAIT 1000
ENDLOOP

LOOP FOREVER
  // infinite loop
  WAIT 5000
ENDLOOP
WHILE / ENDWHILE Control Flow

Loop while a condition is true.

WHILE running == true
  WAIT 1000
  SET running = checkStatus()
ENDWHILE
TRY / CATCH / ENDTRY Control Flow

Error handling. Catch exceptions and handle gracefully.

TRY
  DEF data = CLAW_WEB "https://api.example.com/price"
  BUY 1 AT MARKET
CATCH err
  ALERT "Error occurred" LEVEL "error"
ENDTRY
WAIT Control Flow

Pause execution for a specified number of milliseconds.

WAIT 5000  // wait 5 seconds
ERROR Control Flow

Throw an error, stopping execution (can be caught by TRY/CATCH).

ERROR "Invalid configuration detected"

AI / Analysis Commands

AI_QUERY AI

Query an AI model with a prompt. Optionally specify a tool and argument.

Syntax: 
AI_QUERY <prompt> [TOOL <tool_name>] [ARG <argument>]
AI_QUERY "What is the current market trend for gold?"
AI_QUERY "Analyze sentiment" TOOL "web_search" ARG "bitcoin news today"
DEF analysis = AI_QUERY "Summarize recent trades"
AI_GENERATE_SCRIPT AI

Auto-generate a ClawScript from a natural language prompt.

AI_GENERATE_SCRIPT "Create a momentum strategy using RSI and MACD" TO "momentum_v1"
ANALYZE_LOG AI

Analyze trade or bot logs with AI.

ANALYZE_LOG "What were my best trades this week?" LIMIT 50
DEF insights = ANALYZE_LOG "Show win rate by strategy"
RUN_ML AI

Run a machine learning model on data.

RUN_ML "price_predictor" ON price_data
DEF prediction = RUN_ML "sentiment_model" ON news_feed

Data Fetch Commands

CLAW_WEB Data

Fetch web page content with optional extraction instructions.

CLAW_WEB "https://example.com/data" INSTRUCT "Extract the price table"
DEF page = CLAW_WEB "https://news.example.com"
CLAW_X Data

Search X/Twitter posts.

CLAW_X "bitcoin price prediction" LIMIT 20 MODE "Top"
DEF tweets = CLAW_X "$AAPL earnings" LIMIT 10
CLAW_PDF Data

Extract content from a PDF file.

CLAW_PDF "report.pdf" QUERY "What are the key findings?" PAGES 5
CLAW_IMAGE Data

Generate an AI image from a description.

CLAW_IMAGE "A bullish candlestick chart with green arrows" NUM 2
CLAW_VIDEO Data

Analyze video content from a URL.

CLAW_VIDEO "https://example.com/market-analysis.mp4"
CLAW_CONVERSATION Data

Retrieve conversation history matching a query.

DEF history = CLAW_CONVERSATION "previous trade decisions"
CLAW_TOOL Data

Execute an external tool with optional arguments.

CLAW_TOOL "weather" WITH city = "London"
DEF result = CLAW_TOOL "calculator"
CLAW_CODE Data

Execute a code snippet.

CLAW_CODE "console.log(Date.now())"
DEF output = CLAW_CODE "return prices.slice(-5)"

Agent / Orchestration Commands

SPAWN_AGENT Agent

Create a new agent instance with an optional prompt.

SPAWN_AGENT "analyst" WITH "You analyze market data and provide trade signals"
CALL_SESSION Agent

Call an agent session and get a response.

DEF response = CALL_SESSION "analyst" "What is the trend for EURUSD?"
MUTATE_CONFIG Agent

Change bot configuration at runtime.

MUTATE_CONFIG "stopDistance" = 30
MUTATE_CONFIG "enabled" = false
ALERT Agent

Send an alert notification to a channel.

Syntax: 
ALERT <message> [LEVEL <level>] [TO <channel>] [OPTIONS <opts>]
ALERT "BUY signal on GOLD" LEVEL "info"
ALERT "Stop loss hit!" LEVEL "error" TO "telegram"
SAY_TO_SESSION Agent

Send a message to a specific agent session.

SAY_TO_SESSION "session_123" "Please re-analyze the position"
WAIT_FOR_REPLY Agent

Wait for a reply from an agent session.

DEF reply = WAIT_FOR_REPLY "session_123" TIMEOUT 30 FILTER "signal"

Advanced Commands

CRASH_SCAN Advanced

Enable or disable the crash/flash crash scanner.

CRASH_SCAN ON
CRASH_SCAN OFF
MARKET_NOMAD Advanced

Enable nomadic market scanning — automatically discovers and rotates instruments.

MARKET_NOMAD ON MAX_INSTRUMENTS 4 SCAN_INTERVAL 15
NOMAD_SCAN Advanced

Scan for instruments by category.

DEF instruments = NOMAD_SCAN "forex" LIMIT 10
DEF movers = NOMAD_SCAN "indices" LIMIT 5
NOMAD_ALLOCATE Advanced

Allocate capital to scanned instruments.

NOMAD_ALLOCATE TO instruments SIZING "equal"
RUMOR_SCAN Advanced

Scan for market rumors from various sources.

DEF rumors = RUMOR_SCAN "AAPL" SOURCES "both" LIMIT 20 FILTER "earnings"
OPTIMIZE Advanced

Optimize a parameter over a range of values.

OPTIMIZE rsi_period FROM 7 TO 28 STEP 7 USING 90
INDICATOR Advanced

Calculate a technical indicator by name. Alternative to direct function calls.

INDICATOR RSI
DEF val = INDICATOR(RSI, 14)

Functions Commands

DEF_FUNC / ENDFUNC Functions

Define a reusable function with parameters.

DEF_FUNC checkEntry(rsi, macd)
  IF rsi < 30 AND macd > 0 THEN
    BUY 1 AT MARKET
  ENDIF
ENDFUNC

// Call it
checkEntry(rsiValue, macdValue)
CHAIN Functions

Chain sequential operations, passing results through.

CHAIN fetchData() THEN processData() THEN analyzeResults()
INCLUDE Functions

Include an external ClawScript file.

INCLUDE "common-indicators.cs"

Compiled Strategy Structure

When you compile a ClawScript, the generated JavaScript follows this structure:

'use strict';
const BaseStrategy = require('./base-strategy.cjs');
const indicators = require('../indicators.cjs');
const trade = require('../openclaw-trade.cjs');

class MyStrategyStrategy extends BaseStrategy {
  constructor(config) {
    super(config);
    this._vars = {};
  }

  async evaluateEntry(ticks, context) {
    const config = this.config;
    const prices = ticks.map(t => t.mid || t.close || t.price || 0);
    // ... compiled strategy logic ...
    return null;
  }

  async evaluateExit(position, ticks, context) {
    return { close: false, reason: '' };
  }

  getRequiredBufferSize() { return 100; }
  getDescription() { return 'Custom ClawScript strategy: MyStrategy'; }

  getConfigSchema() {
    return [
      { key: 'enabled', type: 'boolean', default: true, label: 'Enabled' },
      { key: 'size', type: 'number', default: 1, label: 'Position Size' },
      { key: 'stopDistance', type: 'number', default: 20, label: 'Stop Distance' },
      { key: 'limitDistance', type: 'number', default: 40, label: 'Limit Distance' }
    ];
  }

  static get STRATEGY_TYPE() { return 'custom-mystrategy'; }
}

module.exports = MyStrategyStrategy;

Sample: Simple RSI Buy/Sell

A basic strategy that buys when RSI is oversold and sells when overbought.

// Simple RSI Buy/Sell Strategy
// Buys when RSI is oversold, sells when overbought

DEF rsi_period = 14
DEF rsi_oversold = 30
DEF rsi_overbought = 70

DEF rsi = RSI(rsi_period)

IF rsi < rsi_oversold THEN
  BUY 1 AT MARKET STOP 20 LIMIT 40 REASON "RSI oversold entry"
ENDIF

IF rsi > rsi_overbought THEN
  SELL 1 AT MARKET STOP 20 LIMIT 40 REASON "RSI overbought entry"
ENDIF

How it works: Defines RSI with period 14, then uses two IF blocks to check oversold (< 30) and overbought (> 70) conditions, placing market orders with 20-point stops and 40-point limits.

Sample: EMA Crossover with Trailing Stop

Uses fast/slow EMA crossover for entries with trailing stop protection.

// EMA Crossover with Trailing Stop

DEF ema_fast = EMA(9)
DEF ema_slow = EMA(21)
DEF atr = ATR(14)

IF ema_fast CROSSES OVER ema_slow THEN
  BUY 1 AT MARKET STOP 25 REASON "EMA bullish crossover"
  TRAILSTOP 15 ACCEL 0.02 MAX 0.2
ENDIF

IF ema_fast CROSSES UNDER ema_slow THEN
  SELL 1 AT MARKET STOP 25 REASON "EMA bearish crossover"
  TRAILSTOP 15 ACCEL 0.02 MAX 0.2
ENDIF

How it works: The CROSSES OVER / CROSSES UNDER operators detect when the 9-period EMA crosses the 21-period EMA. On crossover, a trade is placed with a trailing stop that accelerates as the price moves in the trade's favor.

Sample: Multi-Indicator with Error Handling

Combines RSI + MACD + Bollinger Bands with try/catch safety.

// Multi-Indicator Strategy with Error Handling

DEF rsi = RSI(14)
DEF macd_hist = MACD(12, 26, 9)
DEF bb_upper = BOLLINGER_UPPER(20, 2)
DEF bb_lower = BOLLINGER_LOWER(20, 2)
DEF price = LAST_PRICE()

TRY
  IF rsi < 30 AND macd_hist > 0 AND price < bb_lower THEN
    BUY 2 AT MARKET STOP 30 LIMIT 60 REASON "Triple confirmation buy"
    ALERT "BUY signal triggered" LEVEL "info"
  ENDIF

  IF rsi > 70 AND macd_hist < 0 AND price > bb_upper THEN
    SELL 2 AT MARKET STOP 30 LIMIT 60 REASON "Triple confirmation sell"
    ALERT "SELL signal triggered" LEVEL "info"
  ENDIF
CATCH err
  ALERT "Strategy error" LEVEL "error"
ENDTRY

How it works: Requires all three indicators to confirm before entering. RSI below 30, positive MACD histogram, and price below lower Bollinger Band must all be true for a buy. The TRY/CATCH block ensures errors (e.g., data unavailable) don't crash the strategy.

Sample: AI Sentiment + Market Scanner

Fetches AI-driven sentiment analysis and combines with technical indicators.

// AI Sentiment + Market Scanner Strategy

DEF rsi = RSI(14)
DEF sentiment = 0

AI_QUERY "Analyze current BTC market sentiment from recent news" TOOL "web_search" ARG "bitcoin sentiment today"
SET sentiment = AI_RESULT

IF sentiment > 0.6 AND rsi < 40 THEN
  BUY 1 AT MARKET STOP 50 LIMIT 100 REASON "Bullish sentiment + RSI dip"
  ALERT "Sentiment BUY entry" LEVEL "info"
ENDIF

IF sentiment < 0.3 AND rsi > 60 THEN
  SELL 1 AT MARKET STOP 50 REASON "Bearish sentiment + RSI high"
  ALERT "Sentiment SELL entry" LEVEL "warning"
ENDIF

CRASH_SCAN ON
WAIT 5000

How it works: Uses AI_QUERY with the web_search tool to analyze real-time BTC sentiment. Combines the sentiment score (0-1) with RSI to find high-conviction entries. Enables crash scanning and adds a 5-second delay between evaluations.

Trade Claw Engine Integration

Compiled ClawScript strategies integrate with the Trade Claw Engine through the strategy loader system:

Strategy Lifecycle

  1. Write: Create ClawScript in the editor
  2. Compile: Click "Compile & Save" to generate JavaScript
  3. Deploy: The .cjs file is saved to skills/bots/strategies/
  4. Load: The strategy loader discovers and registers the strategy
  5. Run: The bot engine calls evaluateEntry() on each tick
  6. Signal: When conditions are met, the strategy returns a trade signal
  7. Execute: The engine places the trade via the IG API
